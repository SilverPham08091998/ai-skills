# leadership/mentoring.md — Mentoring Standard

## Objective

Guide engineers through feedback, pairing, and growth-oriented review.

## When To Use

- Coaching junior/mid engineer
- Reviewing repeated mistakes
- Preparing learning plan

## Scope

- Feedback
- Example
- Why
- Fix
- Practice
- Follow-up

## Tech Lead Rules

1. Give specific feedback with examples.
2. Explain why the rule exists.
3. Suggest concrete fix.
4. Balance quality and encouragement.
5. Turn repeated issues into team guideline.

## Security / Production Rules

1. Do not shame engineers for mistakes.
2. Treat security mistakes as learning plus process improvement.

## Anti-patterns

- Vague feedback.
- Only saying wrong.
- No follow-up.
- Doing everything yourself instead of teaching.

## Expected Output

1. Mentoring comment
2. Learning plan
3. Pairing suggestion
4. Follow-up checklist

## Review Checklist

- [ ] Mentoring comment
- [ ] Learning plan
- [ ] Pairing suggestion
- [ ] Follow-up checklist
